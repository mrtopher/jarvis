# Workflow - /start

Orientation workflow. Explain what this vault is, how it is structured, what the included commands do, and the fastest route to first value.

## Step 1 - Read onboarding docs
Read:
- `START HERE.md`
- `System/Setup Guide.md`
- `System/Command Guide.md`

## Step 2 - Welcome the user
Briefly explain:
- `00 Human/` = their territory
- `Machine/` = AI territory
- `/interview` personalizes the vault
- `/today`, `/new`, `/closeday`, `/meeting-notes` are the core daily operating commands

## Step 3 - Ask up to 2 light questions
Ask one at a time:
1. "What are you using this vault for most right now — business, content, personal life ops, or a mix?"
2. "Do you want the quickest path to value, or a fuller walkthrough?"

## Step 4 - Route the user
Based on the answers, recommend the next move:
- Default next step: `/interview`
- If they want speed: tell them `/interview` is still the right next move because it tailors `/today` and `/closeday`

## Step 5 - Close cleanly
End with a short summary and the exact next command to run.
