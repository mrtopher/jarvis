# Workflow - /content

Turn an idea into finished, on-voice content for one of three channels (LinkedIn post, company blog post, monthly webinar plan), track everything on the content Kanban, and keep it all in the user's voice per `VOICE.md`.

This workflow runs in one of two modes. **Decide the mode first** (Step 0), then run the matching flow. Both flows reuse the shared phases (Context, Draft) defined below.

## Step 0 - Pick the mode
Look at `$ARGUMENTS`:
- **It contains an idea, topic, or text (optionally a channel word like "linkedin", "blog", or "webinar") -> Standalone mode.** Run the **Standalone flow** on that single idea. Ignore the board.
- **It is empty -> Board mode.** Run the **Board flow**, which drives `00 Human/90 Content/Pipeline.md`.

---

## Shared phase: Context
Read these so every piece reflects the user's situation, audience, and voice:
- `Machine/Personalization/content-prompt.md` (compiled content strategy, channel rules, and cadence - regenerated by `/interview`). This is the primary preference source; do not load a stack of separate notes when this exists.
- `VOICE.md` (vault root) - **the authoritative voice for everything written here**. Treat its rules as hard constraints: no em dashes, ever; don't start sentences with "and"; short, punchy sentences. If missing, fall back to `00 Human/70 Context/writing-style.md`.
- `00 Human/70 Context/audience-profile.md` (who the content is for and their pains).
- `00 Human/70 Context/business-profile.md` (background, positioning).
- Recently published pieces in `00 Human/90 Content/{LinkedIn,Blog,Webinars}/` for continuity and to avoid repeating angles.

## Shared phase: Draft (the writing half)
Given an idea and a target channel:
1. **Resolve the channel.** linkedin, blog, or webinar. If the idea text or card carries a channel hint or tag (`#linkedin`, `#blog`, `#webinar`), use it. Otherwise ask the user which channel (skip the question in board mode if the card is tagged).
2. **Pick the template and destination.**
   | Channel | Template | Save to |
   |---------|----------|---------|
   | linkedin | `Machine/Templates/Content LinkedIn Post.md` | `00 Human/90 Content/LinkedIn/YYYY-MM-DD - <slug>.md` |
   | blog | `Machine/Templates/Content Blog Post.md` | `00 Human/90 Content/Blog/YYYY-MM-DD - <slug>.md` |
   | webinar | `Machine/Templates/Content Webinar.md` | `00 Human/90 Content/Webinars/YYYY-MM-DD - <slug>.md` |
   Create the channel folder if it does not exist. Use today's date and a short kebab-case slug from the topic.
3. **Sharpen the angle.** State, in one sentence, the single idea this piece lands and which audience pain it speaks to (per `audience-profile.md`). Tie it to a content pillar from `content-prompt.md` if one fits.
4. **Write the piece.** Fill the template fully in the user's voice per `VOICE.md`. Channel specifics:
   - **LinkedIn:** scroll-stopping hook in the first two lines, short lines, one idea per line, a clear takeaway or question, 3-5 relevant hashtags. No hard sell.
   - **Blog:** scannable subheads, intro hook, body, conclusion + CTA, and a ~155-char meta description. Write for `target_keyword` if set, naturally.
   - **Webinar:** a one-sentence promise, a run-of-show table with rough minutes, key teaching points, a promo plan (pre-webinar LinkedIn + blog tie-in), and a follow-up plan (replay, recap, lead follow-up).
5. **AI-smell and voice pass (remediate in place).** Re-read `VOICE.md`. Scrub: VOICE.md violations (replace every `—`/`–`; no "and"-starts; banned phrases), AI smell (generic buzzwords like "passionate," "leverage synergy," "cutting-edge," "seamlessly," "robust," "dynamic," "in today's fast-paced world"; uniform sentence length; tidy rule-of-three lists), and robotic cadence (vary sentence length). Fix directly in the file.
6. **Suggest repurposing.** Note in the file's repurpose section how this piece can spawn the others (a blog post -> 2-3 LinkedIn posts; a webinar -> a blog recap + promo posts).

## Standalone flow (argument provided)
1. Run **Context**.
2. Run **Draft** on the idea in `$ARGUMENTS`.
3. Add a card to the board: a `- [ ] [[<note path>|<title>]] #<channel>` line under `## Drafting` in `00 Human/90 Content/Pipeline.md` (see **Kanban card mechanics**).
4. Run **Wrap-up**.

## Board flow (no argument)
1. Run **Context**.
2. Read the board at `00 Human/90 Content/Pipeline.md`. Columns are `## Ideas`, `## Drafting`, `## Review`, `## Scheduled`, `## Published` (see **Kanban card mechanics**).
3. **Process every card in `## Ideas`:**
   - Read the idea text and its channel tag. If no channel tag is present, leave the card in place and note it in the report; do not guess the channel.
   - Run the **Draft** phase for that idea and channel.
   - Rewrite the card to link the new note and keep the channel tag, e.g. `- [ ] [[00 Human/90 Content/Blog/2026-06-29 - ai-slop|AI slop is real]] #blog`.
   - **Move the card from `## Ideas` to `## Drafting`.**
4. **Leave `## Drafting`, `## Review`, `## Scheduled`, and `## Published` cards untouched.** From `Drafting` on, the human owns the flow: they edit the draft, move it to `Review` for a second look, to `Scheduled` once approved with a target date, and to `Published` when it ships.
5. If `## Ideas` is empty, tell the user there is nothing to draft and stop.
6. Run **Wrap-up**.

## Shared phase: Wrap-up
1. **Log it.** Append a timestamped entry to today's daily note Activity Log: pieces drafted, channels, and file paths created.
2. **Report back.** Tell the user what was created (vault-relative paths), the angle of each piece, which cards moved and where, and the suggested next step (usually: review the draft, then move the card forward).

## Kanban card mechanics
`Pipeline.md` is an Obsidian Kanban file. Columns are `## <Name>` headings; cards are `- ` (or `- [ ] `) list items under a heading. Channel is marked with a trailing tag: `#linkedin`, `#blog`, or `#webinar`. To **move a card**, delete its line from under the source heading and add it under the target heading, preserving every other line including the trailing `%% kanban:settings ... %%` block. Edit the file with the Edit tool (exact-string replacement), not by rewriting the whole file, to avoid disturbing the settings block.

## Error Handling
| Failure Point | Recovery |
|--------------|----------|
| Channel unclear (standalone) | Ask the user which channel before drafting |
| Idea card has no channel tag (board) | Leave the card in `## Ideas` and report it; do not guess |
| `VOICE.md` missing | Fall back to `00 Human/70 Context/writing-style.md` and tell the user to create `VOICE.md` |
| `content-prompt.md` missing | Run with `audience-profile.md` + `business-profile.md` + `VOICE.md` and tell the user to run `/interview` to compile content preferences |
| Channel folder missing | Create it, then save the note |
